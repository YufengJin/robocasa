# robocasa — Install Recipe

Minimum commands to reproduce this benchmark container from a fresh clone. Generated by `benchmark-env-generator` skill (sub-skill of `env-generator`).

## Host prerequisites

- Docker (>= 24) and the Compose v2 plugin
- NVIDIA driver + NVIDIA Container Toolkit with a GPU visible to `nvidia-smi` (validated on NVIDIA driver 570.x + RTX 40-series; CUDA 11.8 runtime baked into the image)
- Disk budget: ~20 GB for the image plus ~5 GB for kitchen assets on first run. The entrypoint downloads assets the first time only; subsequent starts skip when `robocasa/models/assets/textures/` is populated.

## Build the image

```bash
docker compose -f docker/docker-compose.headless.yaml build
```

Base image: `nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu20.04`
Python: `3.10` (managed by `uv` inside `/opt/venv`)
Third-party simulator pinned: `robosuite @ v1.5.2` (cloned into `/opt/third_party/robosuite`, editable install).

## Bring the container up

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it robocasa_container bash
```

Compose mounts the host checkout (the directory containing `docker/`) to `/workspace` inside the container. The source you edit on the host is the source the interpreter imports — there is no `COPY . .` step.

For an X11 / GUI session (swap the compose profile — container name is shared, only one may run):

```bash
docker compose -f docker/docker-compose.headless.yaml down
xhost +local:docker
docker compose -f docker/docker-compose.x11.yaml up -d
```

## First-run sanity checks (inside the container)

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -c "import robocasa; print(robocasa.__file__)"
```

The last line must resolve under `/workspace/robocasa/` — that proves the mounted source is what the interpreter imports (editable install succeeded).

### Interactive config handled automatically

`entrypoint.sh` writes robosuite's `macros_private.py` once on first boot (via `setup_macros.py`) so that `import robocasa` never blocks on the upstream interactive config prompt. No user action needed.

### Optional asset / dataset download

The entrypoint auto-confirms the kitchen-texture download on first boot (~5 GB into `robocasa/models/assets/textures`). On subsequent starts the download is skipped if the directory already exists. For policy training you will additionally want the Human-50 or Machine-Generated demonstration datasets — see `robocasa/utils/dataset_registry.py` for the HuggingFace URLs; download is **not** done by the entrypoint.

## Category-specific smoke run

### IL: WebSocket random-policy loop

The generated image ships with a companion server so the stack is end-to-end runnable without a trained policy:

```bash
# 1) Start the random-policy server (advertises action_dim=7 for --arm_controller cartesian_pose)
docker exec -d robocasa_container python tests/test_random_policy_server.py --port 8000

# 2) Wait for the port
docker exec robocasa_container python -c "import socket, time
for _ in range(20):
    try: socket.create_connection(('127.0.0.1', 8000), timeout=1).close(); print('ready'); break
    except OSError: time.sleep(0.5)"

# 3) Single-episode demo (writes one MP4 to demo_log_dir)
docker exec robocasa_container python scripts/run_demo.py \
    --policy_server_addr localhost:8000 \
    --task_name PnPCounterToCab --num_resets 1 \
    --demo_log_dir /tmp/smoke_demo

# 4) Eval harness (N episodes, success-rate aggregation)
docker exec robocasa_container python scripts/run_eval.py \
    --policy_server_addr localhost:8000 \
    --task_name PnPCounterToCab --num_trials 1 --no_save_video \
    --log_dir /tmp/smoke_eval

# 5) Clean up
docker exec robocasa_container pkill -f test_random_policy_server
```

The random policy never solves the task — a zero success rate plus a clean exit is the expected outcome.

If port 8000 is already bound on the host (compose uses `network_mode: host`, so container ports share the host namespace), pick a free port with `--port 8001` and pass `--policy_server_addr localhost:8001` to the clients.

docker compose -f docker/docker-compose.headless.yaml down
