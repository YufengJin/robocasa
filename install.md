# robocasa — Install Recipe

Minimum commands to reproduce this container from a fresh clone. Generated by `env-generator` skill.

## Host prerequisites

- Docker (>= 24) and the Compose v2 plugin
- NVIDIA driver >= 470 and the NVIDIA Container Toolkit
- A GPU visible to `nvidia-smi`

Run the bundled check before building:

```bash
bash docker/check_prereqs.sh
```

## Build the image

```bash
docker compose -f docker/docker-compose.headless.yaml build
```

Base image: `nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu20.04`
Python: `3.10` (managed by `uv` in `/opt/venv`)
Install strategy: `uv sync --frozen --no-install-project` (uv.lock pinned)
Third-party simulator: `robosuite v1.5.2` cloned to `/opt/third_party/robosuite` (editable).

## Bring the container up

```bash
docker compose -f docker/docker-compose.headless.yaml up -d
docker exec -it robocasa-headless bash
```

For an X11-enabled session (GUI apps, visualisers):

```bash
xhost +local:docker
docker compose -f docker/docker-compose.x11.yaml up -d
```

Only one of the two profiles may run at a time — they share the container name.

## First-run sanity checks (inside the container)

```bash
nvidia-smi
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -c "import robocasa; print(robocasa.__file__)"
```

The last line must resolve under `/workspace/robocasa/` — this proves that the mounted source is the one the interpreter imports (editable install succeeded).

On first boot the entrypoint downloads kitchen assets (~5 GB) into
`robocasa/models/assets/textures/` and runs `setup_macros.py` to create
`macros_private.py`. Subsequent starts skip both steps when the assets directory
is already populated.

## Stop / clean up

```bash
docker compose -f docker/docker-compose.headless.yaml down
```
